# M3.1 — History and transcripts: durable memory becomes queryable

Depends on: M0.3 (events + summaries already flowing), M1.2 (Slack rendering). Reads:
SPEC §6 (`/v1/history`, transcript endpoint), §7 (session-end), §11.

## Objective

The record the hub has been keeping since M0.3 becomes usable: full transcripts land in the
data dir via the SessionEnd hook, and `/v1/history` + `/agent history` answer "has an agent
touched this repo before, and what happened?" across ALL sessions — done, failed, stopped,
rejected alike. This is the substrate the manager agent (M3.3) reads.

## Deliverables

- `hook/` `session-end` implemented per SPEC §7: parse `transcript_path` from stdin; if
  present, readable, and ≤ 25 MiB, stream it to
  `POST /v1/sessions/{id}/transcript`; oversized ⇒ skip with a stderr note (exit 0 —
  observability never breaks the agent). Unit tests with a stub server: happy path,
  missing file, oversize skip, hub-down exit 0.
- Hub transcript endpoint per §6: session-token-only, raw body streamed to
  `$HUB_DATA_DIR/transcripts/<id>.jsonl` (temp + rename), 25 MiB cap enforced server-side
  too (413), `transcript_stored` event with byte count. Re-upload overwrites (last write
  wins — same session only, so this is safe) — note it in a doc comment.
- `GET /v1/history` per §6: newest-first, `limit` default 50 max 200; `q` is a LIKE over
  summary + recipe_name + repo (case-insensitive); filters composable. Include
  `has_transcript` (file existence) in each row.
- Relay `/agent history [--repo X] [--ticket Y] [free text…]`: ephemeral compact list —
  short, recipe, repo tail, status, age, first 80 chars of summary; footer line with the
  count and active filters. Empty ⇒ friendly "no prior sessions match".
- Provisioning already registers the SessionEnd hook (SPEC §8, since M0.3) — verify, don't
  re-add.
- Tests: history query unit tests (filter combos, LIKE escaping of `%`/`_` in user input,
  ordering, limit clamp); transcript endpoint integration test (auth scoping: session A
  cannot upload to B).

## Steps

1. Transcript endpoint + tests. 2. Hook session-end + tests. 3. History query + tests.
4. Relay command + render tests. 5. Live pass per acceptance. 6. `make check`; PROGRESS.

## Acceptance criteria

1. `make check` green.
2. Live: run two short sessions against the same toy repo (one to completion, one stopped
   early). `ls $HUB_DATA_DIR/transcripts/` shows a `.jsonl` for the completed one (and for
   the stopped one if Claude Code emitted a SessionEnd — record which in PROGRESS);
   `/agent history --repo <toy>` lists BOTH, correct statuses, summaries present where they
   exist.
3. `curl "$HUB/v1/history?q=hello" -H "Authorization: Bearer $HUB_API_TOKEN"` finds the
   session whose summary mentions hello; `q=%` does not match everything (escaping works).
4. Transcript upload with session A's token to session B's path ⇒ 403/404; oversize body ⇒
   413 and no partial file left behind.

## Out of scope

Transcript parsing/summarization/search-inside-transcripts (the manager reads events and
summaries; raw transcripts are for humans until a future milestone), retention policies,
`linked_thread` columns (M1.4's future).

## Stop and ask if

SessionEnd hooks demonstrably don't fire for a mode we rely on (e.g. after `--resume`
phases) — capture evidence and report; do not fake transcripts from stdout.
