# M0.4 — Container spawner (podman) and agent image

Depends on: M0.3. Parallelizable with WS1/WS2 (D5), but REQUIRED before any non-demo use.
Reads: SPEC §8, DECISIONS D5, D10; SETUP A8.

## Objective

`SPAWNER=container` gives every session real isolation: a podman container per phase, the
session workspace and a per-session `.claude` state volume mounted in, the same hook/env
contract as the host spawner, and behavioral parity proven by running the same e2e script
under both spawners. Host bind-mount of `~/.ssh`, `~/.config/gh`, or shared `~/.claude`
(the legacy Model B pattern) is explicitly forbidden here.

## Deliverables

- `images/agent/Containerfile`: debian-slim (or ubuntu) + git + Node 20 + `npm i -g
  @anthropic-ai/claude-code` + the `agentry-hook` release binary copied to
  `/usr/local/bin/agentry-hook`. Build target `make image` (`podman build -t $AGENT_IMAGE
  images/agent`). Document the claude version pin strategy in a comment (pin on build,
  rebuild to upgrade).
- `hub/src/spawn/container.rs` (`ContainerSpawner` implementing the M0.3 trait):
  - Provisioning identical to host (same renderer — refactor, don't duplicate), with
    container-visible paths: workspace dir mounted at `/session` (rw), so settings/mcp/
    policy are `/session/*.json` and cwd is `/session/repo`; `HOOK_BIN` inside the
    container is `/usr/local/bin/agentry-hook` (renderer takes hook path as a parameter).
  - Per-session named volume `agentry-claude-<short>` mounted at `/root/.claude` —
    persists Claude session state across phases so `--resume` works; never shared between
    sessions; retained after session end (history, not garbage).
  - Run one `podman run --rm` per phase via the podman CLI (`--format json` where output is
    parsed): `--name agent-<recipe>-<short>-p<phase>`, `--network host` (hub is
    127.0.0.1-only; record this tradeoff in the module doc — revisit when hub leaves
    localhost), env = SPEC §2 injected set only (assert: no HUBSPOT_*, no host HOME
    leakage), entrypoint = the exact SPEC §8 claude command. Capture stdout for the JSON.
  - `stop` = `podman kill` then `podman rm -f` the phase container; volume untouched.
- Spawner selection at startup from `SPAWNER`; both spawners registered behind the trait.
- `hub/tests/container_cmdline.rs`: unit-level assertions on the constructed `podman run`
  argv (mount set is exactly {workspace, claude-volume}; env allowlist; name scheme) —
  no podman needed for `make check`.
- `scripts/e2e/01_container_parity.sh`: runs `00_direct_session.sh` semantics with
  `SPAWNER=container`, then diffs the observable outcomes (status, HELLO.md committed,
  event kinds present) against a host-spawner run.

## Steps

1. Containerfile + `make image`; manual smoke: `podman run --rm $AGENT_IMAGE claude
--version`. 2. Refactor provisioning renderer for parameterized hook path/prefix (host
milestone tests must stay green). 3. ContainerSpawner + argv tests. 4. Wire selection.
5. Parity e2e live. 6. `make check`; PROGRESS.

## Acceptance criteria

1. `make check` green (no podman required by it).
2. `make image` builds; the smoke command prints a claude version.
3. `scripts/e2e/01_container_parity.sh` passes: container session reaches `done`, commit
   present, `tool_use` events present, and `podman volume ls` shows the session volume.
4. Argv test proves: no mount other than the two specified; env contains only the SPEC §2
   injected set (+PATH/HOME defaults from the image); container name matches
   `agent-<recipe>-<short>-p<phase>`.
5. Two-phase readiness: with a manual `--resume` invocation against the same volume
   (scripted inside the e2e), the second run resumes the first's Claude session (non-empty
   continuation referencing phase-1 context).

## Out of scope

Network egress lockdown (tracked as a follow-up in the module doc), image registry/publishing,
rootful podman, podman REST API (CLI invocation is the accepted mechanism — D5 note).

## Stop and ask if

`--network host` is unavailable in the environment (some rootless setups) — the fallback
(slirp4netns + `host.containers.internal` as AGENTRY_HUB_URL host) changes SPEC §2's injected
URL and needs a SPEC clarification first.
