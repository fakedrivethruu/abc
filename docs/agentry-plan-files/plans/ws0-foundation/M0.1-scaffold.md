## Objective

A compiling, checkable skeleton: `hub/` and `hook/` join the Cargo workspace, `relay/`
exists as a type-checked TypeScript package, `make check` and `make preflight` work, and the
hub serves `/healthz`. No business logic yet.

## Deliverables

- Root `Cargo.toml` edited: `members = ["actor", "hub", "hook"]` (do not touch `actor/`
  contents or the root package for the legacy CLI).
- `hub/Cargo.toml` (package `agentry-hub`) with deps: axum, tokio (full), rusqlite
  (bundled), serde, serde_json, tracing, tracing-subscriber (json), uuid (v4), time
  (formatting), thiserror, anyhow, reqwest (json, no TLS features needed for localhost),
  rand, sha2. Pin current stable versions; no other deps without a milestone listing them.
- `hub/src/main.rs` + `hub/src/config.rs` + `hub/src/http.rs`: load config strictly from
  env per SPEC ¬ß2 (defaults where SPEC gives them; startup error naming any missing required
  var), init tracing JSON, bind `HUB_BIND`, serve `GET /healthz` per SPEC ¬ß6, plus the
  bearer-auth axum middleware skeleton (admin token check on `/v1/*`; 401 with SPEC ¬ß12
  envelope) ‚Äî real routes arrive in later milestones.
- `hook/Cargo.toml` (package `agentry-hook`) deps: serde, serde_json, ureq (json),
  thiserror. `hook/src/main.rs`: subcommand parser (hand-rolled or clap ‚Äî if clap, list it
  in Cargo.toml) recognizing `pretool|posttool|stop|session-end`; each currently exits 1
  with stderr `not implemented until <milestone>` ‚Äî loud absence, not fake success.
- `relay/`: `package.json` (deps: @slack/bolt, typescript, tsx; dev: vitest,
  @types/node), `tsconfig.json` (strict), `src/config.ts` (env loader per SPEC ¬ß2 relay
  vars, throws naming missing vars), `src/index.ts` that loads config, logs
  `relay config ok`, and exits 0. `npm run check` = `tsc --noEmit && vitest run`.
- Root `Makefile`: `preflight`, `check` (exact command chain from CLAUDE.md "Commands" ‚Äî
  clippy/test scoped to `-p agentry-hub -p agentry-hook`, actor excluded), `fmt`,
  `dev-hub` (cargo run -p agentry-hub), `dev-relay` (cd relay && npx tsx src/index.ts),
  `build-hook` (cargo build --release -p agentry-hook).
- `scripts/preflight.sh` per SETUP Part B (STAGE-aware; default STAGE=ws0).
- `.env.example` ‚Äî every SPEC ¬ß2 var with placeholder values and one-line comments.
  Ensure `.gitignore` covers `.env` and `relay/node_modules`.

## Steps

1. Read the existing root `Cargo.toml` and `flake.nix`; confirm adding members doesn't break
   `cargo metadata`. 2. Create crates and relay exactly as above. 3. Write Makefile +
   preflight. 4. Write `.env.example`. 5. Wire config/tracing/healthz/auth middleware.
6. `make check` until green. 7. Update PROGRESS.

## Acceptance criteria (run each; paste results)

1. `make preflight STAGE=ws0` ‚Üí all PASS (with SETUP A1‚ÄìA4 done).
2. `make check` ‚Üí green, including relay.
3. `cargo run -p agentry-hub` starts; `curl -s localhost:8790/healthz` ‚Üí `{"ok":true,‚Ä¶}`;
   `curl -s localhost:8790/v1/anything` ‚Üí 401 with SPEC ¬ß12 envelope;
   same request with `-H "Authorization: Bearer $HUB_API_TOKEN"` ‚Üí 404 (authed, unrouted).
4. `cargo run -p agentry-hub` with `HUB_API_TOKEN` unset ‚Üí exits non-zero naming the var.
5. `target/release/agentry-hook pretool < /dev/null` (after `make build-hook`) ‚Üí exit 1,
   stderr says not implemented.
6. Legacy CLI still builds: `cargo build --bin agentry` succeeds.
